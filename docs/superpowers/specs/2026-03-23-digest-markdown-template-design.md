# 設計: ダイジェスト Markdown の外部 Jinja2 テンプレート化

## 概要

バッチダイジェストの **本文書式**を **コードから分離**し、**Jinja2 テンプレートファイル**を編集するだけでレイアウト・文言・見出し（`#` タイトル含む）を変更できるようにする。テンプレートは **各実行ごとにディスクから読み直す**（再起動不要）。**同梱デフォルト**と **環境変数で指定する外部ファイル**の二層（A+C）とする。

**集約ロジック**（`build_digest_context`）や **DB スキーマ**は本設計の主対象外。変更はテンプレート解決・レンダリング・設定・`run_digest_once` 呼び出し周りに限定する。

---

## 目的

- アプリ再ビルド・再デプロイなしで、**Markdown ダイジェストの体裁**を変更できる。
- 運用で **ファイル名・パス**を環境変数で指定できる。
- **タイトル（H1）**は **テンプレート内**で定義する（コードから `title` を注入しない）。
- **表示用タイムゾーン**を **プロセス単位**で指定し、日時表示をその TZ に合わせられる（集計ウィンドウは従来どおり **UTC の `[from, to)`**）。

## 非目的（本設計のスコープ外）

- LLM の `system` プロンプトや要約形式の外部ファイル化（後続で可）。
- `kind` や API ごとに **別々の表示 TZ** を切り替える（初回は **環境変数 1 本**に固定）。
- テンプレートの管理 UI や DB 保存。

---

## 依存

- **`jinja2`**（`uv add jinja2` を想定）。

---

## 設定（環境変数）

### テンプレート解決

| 変数 | 説明 |
|------|------|
| `DIGEST_TEMPLATE_PATH` | 任意。非空なら **このファイル**を最優先で使用（絶対パス推奨。相対パスは **プロセスのカレントディレクトリ**基準とドキュメント化）。 |
| `DIGEST_TEMPLATE_DIR` | 任意。`PATH` 未指定時、`DIGEST_TEMPLATE_FILE` と組み合わせてパスを構成。 |
| `DIGEST_TEMPLATE_FILE` | 既定例: `digest.md.j2`。`DIR` と組み合わせて使用。 |

**解決順**: `DIGEST_TEMPLATE_PATH` →（未設定または無効なら）`DIGEST_TEMPLATE_DIR` + `DIGEST_TEMPLATE_FILE` → どちらも使えなければ **パッケージ内の同梱デフォルト**（例: `vcenter_event_assistant` 配下の `templates/digest.md.j2` 等、実装時に確定）。

**読み込み**: **`run_digest_once` のたび**にテンプレートファイルを **毎回読み直す**（次回の手動/API/スケジュール実行から反映）。

### 表示用タイムゾーン

| 変数 | 説明 |
|------|------|
| `DIGEST_DISPLAY_TIMEZONE` | 既定: `UTC`。**IANA 名**（例: `Asia/Tokyo`）。**プロセス全体で 1 つ**。無効な名前は **UTC にフォールバック**し、**警告ログ**を出す（検証タイミングは実装時に「起動時」または「初回レンダリング時」のいずれかに統一）。 |

**意味**:
- **集計・`ctx.from_utc` / `ctx.to_utc` の意味**は変えない（**UTC**）。
- **`fmt_ts` フィルタ**は、この設定に従い **表示用に変換**する。

---

## Jinja テンプレートに渡すコンテキスト

| 名前 | 内容 |
|------|------|
| `kind` | `run_digest_once(session, kind=..., ...)` の `kind`（例: `daily`）。H1 等に利用。 |
| `ctx` | `DigestContext.model_dump(mode="json")` に相当する **辞書**。ネスト・日時は JSON 互換（日時は ISO 文字列）。 |
| `display_timezone` | 任意。テンプレートで「表示は ○○ 基準」と注記するため、**解決済み TZ の表示名または文字列**（実装で `ZoneInfo` キー名をそのまま渡すか短いラベルかを決める）。 |

**コードから `title` は渡さない**。見出しはテンプレートで `# ...` を記述する（例: `# vCenter ダイジェスト（{{ kind }}）`）。

### フィルタ `fmt_ts`

- **入力**: ISO 文字列または datetime（どちらでも吸収）。
- **出力**: `DIGEST_DISPLAY_TIMEZONE` に従う **UTC 以外のオフセット付き表示**を既定とする（形式は `YYYY-MM-DDTHH:MM:SS±HH:MM` 等、実装で一貫させる）。
- **登録**: Jinja `Environment` に `fmt_ts` をフィルタとして登録。例: `{{ ctx.from_utc | fmt_ts }}`。

### 件数上限・リスト

- **テンプレート内**で `ctx.top_notable_events[:20]` のようにスライスする方針とする。
- **DB から返る件数の上限**は `digest_context` の定数に依存する。テンプレで 100 行表示したいがクエリが 10 件しか返さない場合は **集約側の変更が必要**である旨をドキュメントに記載する。

## エラー処理

- **テンプレートファイル不在・読み取り不可・Jinja 構文エラー・レンダリング例外**: 集約成功後でも **テンプレート段階で失敗**とみなし、既存方針に合わせ **`DigestRecord.status = error`**、`error_message` に理由。本文は空または短い説明に留める（実装時に統一）。
- **成功時**: 従来どおり LLM を任意で付与。LLM 失敗時の `status=ok` / `error_message` 注記は既存仕様のまま。

---

## データフロー（変更後）

1. `build_digest_context` → `DigestContext`
2. **テンプレート解決**（上記解決順）→ 読み込み → Jinja レンダリング（`kind`, `ctx`, フィルタ `fmt_ts`）
3. `augment_digest_with_llm(..., template_markdown=レンダリング結果)`（既存）
4. `DigestRecord` 挿入

---

## テスト方針

- 一時ディレクトリに `.j2` を置き、`Settings` を上書きしてレンダリング結果に期待文字列が含まれること。
- **同梱デフォルト**テンプレートが読めること。
- **無効な `DIGEST_DISPLAY_TIMEZONE`** で UTC フォールバックとログ（または警告）の挙動。
- **不正テンプレ**で `status=error` になること（必要なら）。

## 承認済みの決定事項（ブレインストーミング）

- 配置: **A+C**（外部ファイル + 同梱デフォルト）、ファイル名は **`DIGEST_TEMPLATE_PATH` または `DIR` + `FILE`** で指定可能。
- 反映: **各実行ごとにディスクから読み直す**。
- エンジン: **Jinja2**。
- タイトル: **テンプレート内**で指定（コードから `title` を渡さない）。
- 表示 TZ: **`DIGEST_DISPLAY_TIMEZONE`（プロセス単位 1 本）**で問題なし。
