# ダイジェスト利用者向けドキュメント — 設計

**日付:** 2026-05-23  
**ステータス:** 承認済み（実装に反映）  
**正本（利用者向け）:** [`docs/user-guides/digests.md`](../../user-guides/digests.md)

## 目的

期間集計レポート（**ダイジェスト**）の意味・画面操作・LLM 要約の有無・他機能との関係を、[`alerts.md`](../../user-guides/alerts.md) / [`score-rules.md`](../../user-guides/score-rules.md) と同型で利用者向けにまとめる。

## スコープ

### 含める

- ダイジェストとは何か（日次・週次・月次、サーバー保存）
- 生成の流れ（集計テンプレート → 任意で LLM 要約）
- 本文の読み方（テンプレート節の意味、要注意 ≥40、種別集約）
- **ダイジェスト** タブの操作（種別フィルタ、一覧、本文、Markdown ダウンロード）
- 定期生成・手動生成は **管理者／API** 向けに概要のみ（`.env` は backend / getting-started へ）
- チャット・スコアルール・概要との違い
- FAQ

### 含めない

- Jinja テンプレートのカスタマイズ手順の詳細（backend / `.env.example`）
- 画面からの手動生成 UI（未実装）
- 英語版
