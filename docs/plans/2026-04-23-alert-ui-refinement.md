# アラート関連 UI のデザイン統一プラン (2026-04-23)

## 目的
「通知履歴」メインタブおよび「設定 > アラート」サブタブのボタン・レイアウトを、既存の他のタブ（ダイジェスト、vCenter 設定等）と共通のデザインに統一し、ユーザー体験を一貫させる。

## 修正内容

### 1. 通知履歴 (AlertHistoryPanel)
- [ ] コンポーネントのルートクラスを調整し、共通のパネルマージンを適用。
- [ ] ヘッダー構造を `DigestsPanel` と同様の形式に変更。
- [ ] 右上のボタンラベルを「一覧を更新」にし、`className="primary"` を適用。
- [ ] テーブルを囲む「枠（コンテナ）」のスタイルを共通化。

### 2. アラート設定 (AlertRulesPanel)
- [ ] ヘッダーおよび「新規ルール追加」ボタンのスタイルを `VCentersPanel` 等と統一。
- [ ] ルール一覧の各項目（カード）の余白や境界線を他の設定項目と合わせる。
- [ ] フォーム内の入力要素やボタンのレイアウトを調整。

### 3. スタイルシートの整理
- [ ] `AlertHistoryPanel.css` および `AlertRulesPanel.css` 内の独自定義を削減し、`App.css` の共通クラスを優先して使用。

## 完了条件
- [ ] 通知履歴タブを表示し、ダイジェストタブとボタン・ヘッダーが同じ見た目であること。
- [ ] 設定 > アラートタブを表示し、他の設定サブタブ（vCenter 等）とカードやボタンのデザインが同じであること。
