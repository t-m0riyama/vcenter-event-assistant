# 要件ID-テスト証跡マッピングテンプレート

## 使い方
- プランの要件ごとに `REQ-xx` を付与して記録する。
- 1要件につき最低1つの検証手段（自動テスト推奨）を紐付ける。
- PR前に `結果` を必ず更新する。

## マッピング表
| 要件ID | 要件概要 | 判定基準 | 対応テスト/確認手順 | 結果 | 備考 |
|---|---|---|---|---|---|
| REQ-01 | `<要件概要>` | `<期待挙動>` | `frontend/src/...test.ts` の `it('...')` | 未実施 |  |
| REQ-02 | `<要件概要>` | `<期待挙動>` | `pytest tests/... -k ...` | 未実施 |  |

## 結果の記入ルール
- `結果` は `PASS / FAIL / N/A` のみ使用する。
- `FAIL` の場合は原因と次アクションを `備考` に記載する。
- 手動確認を使う場合は、再現手順を `対応テスト/確認手順` に具体記載する。
