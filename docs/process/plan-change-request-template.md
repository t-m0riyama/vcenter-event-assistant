## 記入例
```md
## 申請情報
- 申請日: 2026-05-07
- 申請者: moriyama
- 対象プラン: docs/plans/2026-05-07-incident-timeline-mvp-plan.md
- 対象要件ID: REQ-02

## 変更内容
### 現行要件
統合タイムライン（横軸=時刻）

### 提案要件
MVPでは縦積み表示を採用し、横軸化は次フェーズで対応する

### 変更理由
- 実装済みUIの改修コストが高く、リリース期限に収まらない
- 先行リリースで運用フィードバックを取りたい

## 影響範囲
- コード: frontend/src/panels/chat/IncidentTimelinePanel.tsx, frontend/src/panels/chat/ChatPanel.css
- テスト: frontend/src/panels/chat/IncidentTimelinePanel.test.tsx
- API/仕様: なし
- UI挙動: あり
- 後方互換性: 維持

## 代替案
1. 既存どおり横軸へ改修（メリット: 仕様一致 / デメリット: 工数増）
2. MVPは縦積み維持（メリット: 早期リリース / デメリット: 仕様差分を管理必要）

## リスクと緩和策
- リスク: 仕様と実装が乖離する
- 緩和策: Plan Compliance節に差分を明示し、次フェーズの要件としてチケット化

## 承認
- 承認者: tech-lead
- 承認日時: 2026-05-07 16:20
- 承認コメント: MVPは縦積みで可。次フェーズで横軸化を必須とする
```
