# プラン準拠チェックリスト

## 使い方
- 対象: `docs/plans/*.md` に基づく実装作業
- タイミング: 着手前、実装中、PR前
- ルール: 各項目は `完了 / 未完了 / 非該当` のいずれかを必ず記録する

## 0. 基本情報
- プラン: `<path>`
- 作業ブランチ: `<branch>`
- 担当: `<name>`
- 更新日: `<yyyy-mm-dd>`

## 1. 着手前ゲート
- [ ] 要件に一意なIDを付与した（例: `REQ-01`）
- [ ] 各要件の判定方法を明記した（テスト名または確認手順）
- [ ] 変更禁止領域を明記した（互換性、既存挙動、除外範囲）
- [ ] 不明点を解消した（未解消なら実装を開始しない）
- [ ] 要件ごとにREDテストを先に追加した

## 2. 実装中ゲート
- [ ] 要件単位で実装を分割した（1要件ずつ Green 化）
- [ ] 要件に対応しない実装変更を含めていない
- [ ] 逸脱の可能性が出た時点で実装を停止した
- [ ] 逸脱時は先に変更申請テンプレを起票した
- [ ] 合意済み変更のみ実装した（合意リンクを残した）

## 3. PR前ゲート
- [ ] 要件IDと対応テストの対応表を更新した
- [ ] 対象テストを実行し、結果を記録した
- [ ] 回帰テストを実行し、結果を記録した
- [ ] PR本文に Plan Compliance 節を貼り付けた
- [ ] 未対応要件と理由を明記した（未対応がない場合は `なし`）

## 4. 逸脱判定ルール
次のいずれかに該当する場合、プラン逸脱とみなす。
- プラン未記載の挙動変更を行った
- プラン記載要件を満たしていない
- 合意なく要件を追加または削除した

## 5. 記録テンプレ
```md
### チェック結果
- 実施日: 2026-05-07
- 判定: 完了
- 未完了項目: なし
- 備考: 仕様変更なし
```
