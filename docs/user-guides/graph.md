# グラフタブの使い方（利用者向け）

vCenter Event Assistant の **グラフ** タブは、取り込み済みの **ホストメトリクス**（CPU/メモリ利用率など）を時系列で表示します。任意で **イベント件数** を同じグラフに重ねられます。本書は開発者向けではなく、負荷の推移確認とエクスポート操作を把握したい運用担当者向けに、現行の挙動をまとめたものです。

---

| あなたの役割 | まず読む節 | 目的 |
|-------------|-----------|------|
| **運用（監視）** — 負荷の推移を見る | [§4](#4-グラフの読み方) → [§7](#7-画面別操作) | 軸・系列の意味 |
| **運用（障害）** — イベントと同時刻を比較 | [§5](#5-イベント件数のオーバーレイ) | 種別指定の使い方 |
| **運用（報告）** — 図表を残す | [§7](#7-画面別操作) のエクスポート | SVG / CSV |

用語の区別:

| 用語 | 意味 |
|------|------|
| **メトリクスキー** | 収集している指標の種類（例: `host.cpu.usage_pct`）。プルダウンで選択。 |
| **表示期間** | グラフに載せる時系列の範囲（タイムゾーンは **設定 → 一般**）。 |
| **サンプル** | 一定間隔で DB に保存された計測点（サーバー設定の取得間隔に依存）。 |
| **条件一致** | 現在の vCenter・メトリクスキー・期間に合う DB 上のサンプル総数。 |

---

## 2. グラフタブの全体像

1. スケジュール等で **メトリクス** が vCenter から取り込まれ DB に蓄積される。
2. **グラフ** タブで vCenter・メトリクスキー・表示期間を指定する（初回は **直近 24 時間** のプリセット）。
3. API がサンプルを返し、折れ線グラフに描画する（`host.*` 系は **ホストごと** に系列分割）。
4. 必要なら **イベント種別** を指定し、右軸に **イベント件数** を重ねる。
5. **CSV / SVG** で持ち出したりできる。

```
[vCenter 取り込み（メトリクス）]
        ↓
[DB にサンプル蓄積]
        ↓
[グラフタブ] ← vCenter・キー・期間
        ↓
    ┌───┴───┬────────────┐
    ↓       ↓            ↓
[イベント重ね] [再取得] [エクスポート]
```

メトリクスは vCenter **quickStats** 等に基づく **限定的な項目** です。全カウンタ網羅や VM 単位の詳細分析専用ではありません（README の制約を参照）。

## 3. ツールバーと表示期間

### 常時表示

| 項目 | 説明 |
|------|------|
| **vCenter** | **全て** または登録済み 1 台。ホスト別系列の対象が変わる。 |
| **メトリクスキー** | 表示する指標。DB に未保存のキーも、既知一覧に含まれていれば選択可能（例: `host.cpu.usage_pct`）。 |
| **イベント種別（任意・完全一致）** | §5。空欄ならメトリクスのみ。 |
| **再取得** | 条件を変えず API から読み直す（グラフの再描画キーも更新）。 |
| **グラフをダウンロード** | 表示中のグラフを **SVG** で保存。 |
| **CSV をダウンロード** | 表示中のサンプル点を **CSV** で保存。 |
| **条件一致: N 件（表示: M 件まで）** | DB 上の該当サンプル数と、グラフに載せた点数。 |

### 表示期間（折りたたみ）

- **設定 → 一般** の **タイムゾーン** 上の日時として解釈されます。
- **開始・終了を両方指定** するか、**両方空** にしてください。**片方だけ** の入力は **エラー** になります（**イベント** タブの期間指定とは異なります）。
- **日付のみ**（時刻空欄）… 開始 **0:00**、終了 **23:59**。
- **クイック**（過去 24 時間 / 2 日 / 7 日 / 30 日）… 直近のローリング期間を一括設定（既定は **24 時間**）。
- 期間を指定したとき、メトリクスは最大 **10,000** 点まで取得します。期間 **未指定**（両方空）のときは直近 **500** 点です。
- **自動更新（N 分ごと）** チェック … **設定 → 一般** の間隔（既定 5 分）で、**グラフ** タブ表示中に再取得します。オフにすると手動の **再取得** のみ。

## 4. グラフの読み方

### 左軸（メトリクス）

- 折れ線は選択した **メトリクスキー** の値。
- キーが `*_pct` で終わる場合、左軸ラベルは **％** です。
- **`host.*` / `datastore.*`** キーは、**エンティティ（ホスト・データストア）ごと** に色分けされた複数系列になります。
- それ以外のキーは **1 本** の系列（`v`）です。
- 凡例を **クリック** すると、その系列の表示／非表示を切り替えられます。

### 横軸

- **時刻**（タイムゾーン表示）。データの範囲に合わせて自動調整されます。

### データが無いとき

画面にヒントまたは空状態が出ます。

- メトリクスキーが DB に無い … 取り込み待ち・vCenter 無効・接続不良を確認。
- **該当するメトリクスがありません（条件一致 0 件）** … 期間を広げる、vCenter の **有効**、接続テストを確認。

## 5. イベント件数のオーバーレイ

- **イベント種別** に **完全一致** の文字列（例: `vim.event.VmPoweredOnEvent`）を入れると、同じ表示期間でその種別の発生 **件数** を、メトリクス取得間隔と同じ幅の **時間バケット** に集計し、**右軸（イベント件数）** に折れ線で重ねます。
- 入力欄の **datalist** から、DB に存在する種別の候補を選べます（vCenter 絞り込みに追従）。
- **空欄** のときはメトリクスのみ（右軸・イベント系列なし）。
- バケット幅は画面に表示される **メトリクス取得間隔（秒）**（サーバー設定。例: 300 秒）と同じです。

イベントの行単位の一覧は **イベント** タブ（[events.md](events.md)）で確認してください。

## 6. 他機能との関係

| 機能 | グラフとの違い |
|------|----------------|
| **概要** | 直近 24h の **ピーク上位表**。グラフは **任意期間の時系列**（[summary.md](summary.md)）。 |
| **イベント** | イベントログの一覧。グラフのオーバーレイは **件数の推移** のみ。 |
| **タイムライン** | 期間内の統合タイムライン。メインタブから切り替えて利用する。スナップショットから **グラフ** タブへ再生できる（§6 参照）。 |
| **アラート（メトリクス閾値）** | メール通知。アラートの `metric_key` はグラフのキーと **完全一致** させる（[alerts.md](alerts.md)）。 |
| **チャット** | 期間メトリクスを LLM に送る別経路（[chat.md](chat.md)）。 |
| **ダイジェスト** | 期間内ホスト CPU/メモリ **上位表** の抜粋（[digests.md](digests.md)）。 |

### タイムラインのスナップショットからの再生

**タイムライン** で保存したスナップショットを、**グラフ** タブで再現表示することがあります（期間・vCenter・メトリクス・種別・縦線マーカー）。通常の手動操作と同じ画面で確認できます。

## 7. 画面別操作

### グラフ（メインタブ）

画面例: [グラフタブ](../images/metrics.png)

1. **vCenter** と **メトリクスキー** を選ぶ（初回は先頭の vCenter とキーが自動選択されやすい）。
2. **表示期間** を確認・変更する（既定は **直近 24 時間**）。
3. グラフが表示されたら、ツールチップで時刻・値を確認する。
4. 必要なら **イベント種別** を入れて右軸の件数線を表示する。
5. **再取得** で最新サンプルを取り込む（**自動更新** オン時は間隔どおり）。
6. 報告用に **グラフをダウンロード** または **CSV をダウンロード**。

## 8. よくある質問と切り分け

### グラフが真っ白／キーが選べない

- vCenter が未登録・無効、またはまだメトリクスが DB に無い。数分後に **再取得**。
- **メトリクスキーがありません** … 取り込み前でも既知キーは選べますが、**点が無い** と空表示になります。

### 期間を変えたらエラーになる

- **開始・終了の片方だけ** 入力していないか確認（両方指定 or 両方空）。
- 存在しない時刻（サマータイムの欠落時刻など）を指定していないか。

### アラートの CPU とグラフの値が合わない

- アラートの **メトリクスキー** がグラフで見ているキーと **完全一致** しているか（例: `host.cpu.usage_pct`）。旧 UI 既定名との取り違えに注意（[alerts.md](alerts.md) §8）。

### 表示件数が「10000 件まで」で足りない

- 期間を短くするか、vCenter を 1 台に絞る。長期の全点表示は API 上限のため分割検討。

### イベントの線が出ない

- **イベント種別** が空でないか、表記が **完全一致** か、期間内にその種別のイベントがあるか。

### 自動更新が動かない

- **表示期間** 内の **自動更新** チェックと、**設定 → 一般** の **自動更新** がオンか。**グラフ** タブを表示している必要があります。
