# チャット機能の使い方（利用者向け）

vCenter Event Assistant の **チャット** は、収集済みの **イベント** と（任意で）**ホストメトリクス** を、指定した **集計期間** のコンテキストとして LLM に渡し、自然言語で調査・整理を手伝う機能です（README 上は **ベータ**）。本書は開発者向けではなく、画面操作とデータの扱いを把握したい運用担当者向けに、現行の挙動をまとめたものです。

---

| **運用（調査）** — 障害時に期間を絞って質問する | [§3](#3-チャットで送られる情報) → [§7](#7-画面別操作) | 何が LLM に渡るかと操作手順 |
| **運用（定型化）** — よく使う質問を共有したい | [§8](#8-サンプルの質問設定--チャット) | スニペットの編集・JSON バックアップ |
| **管理者** — LLM を有効にする | [§5](#5-llm-とサーバー側の前提管理者向け概要) → [getting-started.md](../getting-started.md) | API キー・プロバイダ設定 |

用語の区別:

| 用語 | 意味 |
|------|------|
| **集計期間** | チャット上部で指定する分析対象の時間範囲。DB 上は UTC の **半開区間** `[開始, 終了)`。表示は **設定 → 一般** のタイムゾーンに従う。 |
| **コンテキスト** | その期間からサーバーが組み立て、LLM に渡す集約データ（イベント統計・要注意イベント・任意のメトリクスなど）。 |
| **会話** | あなたとアシスタントのメッセージ履歴。**このブラウザの localStorage** に保存され、**サーバーには保存されない**。 |
| **プレビュー** | LLM を呼ばず、送信直前にサーバーが組み立てたプロンプト内容を確認する機能。 |

## 2. チャット機能の全体像

1. **イベント** と **メトリクス** を vCenter から取り込んでおく（前提）。
2. チャット画面で **集計期間**・**対象 vCenter**・（任意）**期間メトリクス** を指定する。
3. 質問を送信すると、サーバーが期間内データを集約し、**直近の会話** と合わせて LLM に送る。
4. 回答は画面に表示される。同じ期間で **追質問** できる。
5. **会話・下書き・サンプル質問** はブラウザに保持される。別 PC・別ブラウザでは共有されない。

```
[取り込み済みイベント・メトリクス]
        ↓
[集計期間 + vCenter + オプションでメトリクス]
        ↓
[サーバーがコンテキスト JSON を組み立て]
        ↓
[LLM] ← 直近会話（最大 20 ターン分を送信）
        ↓
[回答を画面表示]（会話はブラウザに保存）
```

**ダイジェスト** は同じデータ源でも **保存済みレポートの閲覧** が中心です。**チャット** は **任意期間を対話で深掘り** する用途向けです（[digests.md](digests.md) §6）。

## 3. チャットで送られる情報

### 常に含まれるもの（イベント）

チェックボックスはありません。送信のたびに、おおむね次がコンテキストに入ります。

- 期間内の **イベント総数**、**要注意（notable_score ≥ 40）件数**
- **イベント種別ごとの件数**（上位）
- **要注目スコアの高いイベント**（種別ごとに集約したグループ。上限あり）
- 期間メトリクスを **1 つでもオン** にしたときは、時間バケットごとの **イベント件数推移** も追加

スコアの意味・調整は [score-rules.md](score-rules.md) を参照してください。

### 任意（期間メトリクス）

**LLM に含めるメトリクス** のチェックをオンにしたカテゴリだけ、期間内を **バケット平均** した系列を追加します（**追加の DB クエリ** があり、期間が長いと重くなり得ます）。

| チェック | 内容 |
|---------|------|
| **CPU 使用率** | ホスト CPU 利用率の時系列 |
| **メモリ使用率** | ホストメモリ利用率の時系列 |
| **ディスク IO** | ディスク IO 関連 |
| **ネットワーク IO** | ネットワーク IO 関連 |

### メトリクス閾値（%）

その下の **CPU / Memory / Disk / Network 閾値** は、コンテキスト内の **インシデントタイムライン** 組み立てに使う目安です（タイムラインタブの考え方に近い）。メトリクスチェックがオフでも値は送られますが、メトリクス系列が無いと効果は限定的です。

### 対象 vCenter

- **すべて（登録済み全体の集約）** … 登録済み vCenter 全体。
- 特定 vCenter を選択 … その接続先に絞った集約。

## 4. 会話の保持と制限

| 項目 | 挙動 |
|------|------|
| **保存場所** | **ブラウザの localStorage**（キー `vea.chat_panel.v1` など）。**サーバーは会話を保存しない**。 |
| **最大保持件数** | **設定 → 一般** の **チャットの最大保持件数**（0〜1000、既定 **200**）。超えた分は **古いメッセージから削除**（FIFO）。0 は保持なし（送信は可能）。 |
| **LLM に送る会話** | 1 回の API 呼び出しでは、直近 **20 件** のメッセージまで（表示上 200 件あっても送信は 20 件に収まる）。 |
| **下書き** | 入力中テキストもブラウザに保存される（短い遅延後）。 |
| **別端末** | 会話・下書き・サンプルは **共有されない**。 |

送信後、画面下部に **LLM 入力（目安）** が出ることがあります（推定トークン数、JSON 切り詰めの有無、会話ターン数）。上限に近いときは期間を短くするか、メトリクスチェックを減らしてください。

## 5. LLM とサーバー側の前提（管理者向け概要）

- チャットを動かすには、サーバーに **LLM が設定** されている必要があります。未設定時は送信が **503** となり、画面上部にエラーが出ます。
- 環境変数は主に **`LLM_DIGEST_*`**（ダイジェストと共通）と、任意の **`LLM_CHAT_*`**（チャットだけ上書き）です。詳細は [getting-started.md](../getting-started.md) と `.env.example`。
- **`LLM_ANONYMIZATION_ENABLED`**（既定オン）のとき、外部 LLM へはホスト名などがマスクされ、**回答表示時にサーバーで実名に戻す** ことがあります。送信内容の取り扱いは管理者のポリシーに合わせてください。
- 本機能は **ベータ** です。回答は補助資料とし、**唯一の公式記録** にしない運用を推奨します（README の制約も参照）。

## 6. 他機能との関係

| 機能 | チャットとの違い |
|------|------------------|
| **ダイジェスト** | サーバーに保存された **定型 Markdown レポート** の閲覧。チャットは **対話** と **任意期間**。 |
| **タイムライン** | 視覚的な時系列。チャットは **言語での要約・比較・手順の相談**。 |
| **イベント** | 行単位の一覧・フィルタ（[events.md](events.md)）。チャットは **集約済みコンテキスト** ベース。 |
| **アラート** | メール通知。チャットは **通知を送らない**（[alerts.md](alerts.md)）。 |
| **概要** | **直近24h** のダッシュボード（[summary.md](summary.md)）。チャットは **自分で指定した期間**。 |

### 活用例

- **障害直後**: 発生時刻前後を **過去 24 時間** などに設定し、「CPU 利用率のピークと同時刻のイベント種別の関係を整理して」と質問する。
- **週次レビュー**: **過去 7 日** でメトリクスをオンにし、ダイジェストを読んだうえで「再発防止を優先すべき事象を 3 つに絞って」と追質問する。

## 7. 画面別操作

### チャット（メインタブ）

画面イメージは [チャット画面](../images/chat.png) を参照してください。

#### 上部（コンテキストの指定）

1. **集計期間** … 開始・終了の日時。**クイック**（過去 24 時間 / 2 日 / 7 日 / 30 日）でプリセットも使える。
2. **対象 vCenter** … 全体または 1 台に絞る。
3. **LLM に含めるメトリクス** … 必要なカテゴリだけオン（§3）。
4. **メトリクス閾値（%）** … 必要に応じて調整。

#### 会話エリア

- **あなた** / **アシスタント** の吹き出し。アシスタントは Markdown 表示。
- **回答をコピー** … アシスタント行のボタンでクリップボードへ。
- 送信中は「応答を生成しています…」と表示。

#### 下部

| 操作 | 説明 |
|------|------|
| **会話をクリア** | ブラウザ上の会話をすべて削除（確認あり）。 |
| **サンプルの質問**（チップ） | クリックで **下書き末尾に追記**（**送信はしない**）。繰り返しクリックで追記が続く。 |
| **メッセージ** | 質問入力。**Enter** で送信、**Shift+Enter** で改行。 |
| **プレビュー** | LLM を呼ばず、送信予定のコンテキスト・会話をモーダルで確認。 |
| **送信** | 質問を送り回答を得る。 |

期間が未入力・不正のときは送信前にエラーになります。

## 8. サンプルの質問（設定 → チャット）

**設定 → チャット** で、チャット下部に出る **サンプルの質問** チップを編集します。

- **保存先** … **このブラウザの localStorage**（サーバー非共有）。
- **ラベル** と **質問本文** の両方に文字がある行だけがチップとして表示される。
- **サンプルを追加** / 行の削除 / **既定に戻す**（同梱の初期セットに復元、確認あり）。
- **ファイルにエクスポート** / **ファイルからインポート** … JSON（`vea-chat-sample-prompts` 形式）。インポート時の上書き・ファイル外 id 削除オプションに注意（スコアルールと同様の確認ダイアログ）。

設定画面: [チャット設定](../images/settings-chat.png)

## 9. よくある質問と切り分け

### 送信すると「LLM が未設定」と出る

- サーバー管理者に **`LLM_DIGEST_API_KEY`** または **`LLM_CHAT_API_KEY`**（または Copilot CLI セッション認証）の設定を依頼してください（§5）。

### 回答が遅い・途中でエラーになる

- 期間を短くする、メトリクスチェックを減らす、会話を **クリア** してターン数を減らす。
- **LLM 入力（目安）** で JSON 切り詰めが出ている場合は、データ量が上限に近い状態です。

### ブラウザを変えたら会話が消えた

- 正常です。会話は **端末・ブラウザごとの localStorage** のみです（§4）。

### サンプルチップを押しても送信されない

- 仕様です。追記後に内容を直し、**送信** を押してください（§7）。

### ダイジェストと同じ内容を毎回聞かれる

- チャットは **その時点の DB** から集計します。ダイジェストは **生成時点のスナップショット** です。期間・スコアルール変更後は、チャットの方が新しい集計になります。

### イベントのチェックボックスがない

- イベント集約は **常にオン** です。増やしたいのは **期間メトリクス** のチェックです（§3）。

### プレビューと本番の回答が違う

- プレビューは **プロンプト確認用** で、LLM 回答は生成しません。送信時点の下書き・会話が同じでも、LLM の応答は確率的に変わり得ます。
