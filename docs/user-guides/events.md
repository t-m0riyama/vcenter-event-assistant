# イベントタブの使い方（利用者向け）

vCenter Event Assistant の **イベント** タブは、取り込み済みの **vCenter イベント** を一覧し、期間やスコア・キーワードで絞り込み、**運用メモ** を付けたり **CSV** で持ち出したりするための画面です。本書は開発者向けではなく、調査・記録の日常操作を把握したい運用担当者向けに、現行の挙動をまとめたものです。

---

## 1. この文書の対象と読み方

| あなたの役割 | まず読む節 | 目的 |
|-------------|-----------|------|
| **運用（調査）** — 障害時にイベントを追う | [§4](#4-一覧の列の見方) → [§7](#7-画面別操作) | 絞り込みと表の読み方 |
| **運用（記録）** — 調査メモを残す | [§5](#5-運用メモ) | コメントの付け方 |
| **運用（報告）** — データを外部へ | [§7](#7-画面別操作) の CSV | エクスポートの範囲 |

用語の区別:

| 用語 | 意味 |
|------|------|
| **イベント** | vCenter から取り込まれた 1 件のログ行（種別・時刻・メッセージなど）。 |
| **要注目スコア** | 重要度 0〜100（表の **スコア** 列）。[score-rules.md](score-rules.md) |
| **重大度（severity）** | VMware 由来の分類（**スコアとは別**）。 |
| **運用メモ** | このアプリ上でイベント行に付ける `user_comment`（サーバーに保存）。 |

---

## 2. イベントタブの全体像

1. vCenter からイベントが **収集** され DB に蓄積される。
2. **イベント** タブで条件を指定し `GET /api/events` が一覧を返す（**新しい順**）。
3. 行の **運用メモ** は `PATCH /api/events/{id}` で **サーバー** に保存される（全ブラウザ共通）。
4. **CSV をダウンロード** すると、**現在の絞り込み条件に一致する全件** をまとめて取得できる。
5. ざっと見る用途は **概要** タブ、対話分析は **チャット** タブへ。

```
[vCenter 取り込み]
        ↓
[DB にイベント蓄積]
        ↓
[イベントタブ] ← 期間・スコア・テキスト絞り込み
        ↓
    ┌───┴───┬────────────┐
    ↓       ↓            ↓
[運用メモ保存] [CSV出力] [概要・チャット・グラフ]
```

## 3. 絞り込みの考え方

### ツールバー（常時表示）

| 項目 | 説明 |
|------|------|
| **最小スコア** | 要注目スコアの **下限**（数値）。空欄は **下限なし**。例: `40` でスコア 40 以上のみ。 |
| **表示件数** | 1 ページあたり **20 / 50 / 100 / 200** 件（既定 50）。 |
| **前へ / 次へ** | ページ送り。 |
| **全 N 件中 X–Y 件を表示** | 条件に一致する総件数と、現在ページの範囲。 |
| **CSV をダウンロード** | §7。件数 0 のときは無効。 |

### 絞り込み条件（折りたたみ）

**絞り込み条件** を開くと、次を指定できます。

#### 表示期間

- **設定 → 一般** の **タイムゾーン** 上の日時として解釈されます。
- **開始・終了とも空** … 期間による制限 **なし**（DB にあるイベントが対象。件数が多い環境では注意）。
- **片方だけ入力** … もう片方は **制限なし**（開始のみ＝その時刻以降、終了のみ＝その時刻以前）。
- **日付だけ**（時刻空欄）… 開始はその日 **0:00**、終了はその日 **23:59** として扱います。
- **クイック**（過去 24 時間 / 2 日 / 7 日 / 30 日）で期間を一括入力できます。

#### テキスト絞り込み（いずれも **部分一致**、大文字小文字は区別しない）

| 項目 | 対象列 |
|------|--------|
| **種別（含む）** | `event_type` |
| **重大度（含む）** | `severity` |
| **メッセージ（含む）** | `message` |
| **コメント（含む）** | **運用メモ** 本文 |

見出し右のプレビュー文字列で、現在のテキスト条件の要約が分かります。

条件を変えると **1 ページ目から** 再検索されます。編集中の運用メモは **ページや条件変更でキャンセル** されます。

## 4. 一覧の列の見方

| 列 | 内容 |
|----|------|
| **時刻** | イベント発生時刻（一般設定のタイムゾーン表示）。 |
| **種別** | `event_type`（例: `vim.event.VmPoweredOnEvent`）。 |
| **ガイド** | **表示** で [イベント種別ガイド](../event-type-guides/README.md) を展開（意味・対処など）。未登録は `—`。 |
| **重大度** | VMware の severity。 |
| **スコア** | 要注目スコア。 |
| **メッセージ** | vCenter 由来の本文。 |
| **運用メモ** | §5。 |

- 一覧の並びは **時刻の新しい順**（`occurred_at` 降順）。
- 種別ガイドで **対処が必要** の行は、背景が強調表示されます（概要タブと同じ）。

---

## 5. 運用メモ

- **運用メモ** は調査メモ・チケット番号・対応状況など、運用者が自由に書くテキストです。
- 行末の **編集** → テキストエリア（最大 **8000** 文字）→ **保存** / **キャンセル**。
- 空にして保存するとメモは **削除**（`null`）されます。
- 保存先は **サーバー（DB）** です。**概要** タブの同じ列は **参照のみ**（[summary.md](summary.md)）。
- 他の利用者や別ブラウザからも、同じイベントを開けば同じメモが見えます。

## 6. 他機能との関係

| 機能 | イベントタブとの違い |
|------|---------------------|
| **概要** | **直近 24h** の抜粋（統計・上位 10 件など）。イベントは **任意期間・全件検索**（[summary.md](summary.md)）。 |
| **グラフ** | ホストメトリクスの時系列（[graph.md](graph.md)）。 |
| **チャット** | 自然言語での期間分析。一覧の代わりに **集約 JSON** を LLM へ（[chat.md](chat.md)）。 |
| **ダイジェスト** | 保存済みレポート。イベントタブは **生の行**（[digests.md](digests.md)）。 |
| **アラート** | メール通知。イベント一覧は通知の有無に関係なく **全取り込み分** を検索（[alerts.md](alerts.md)）。 |
| **スコアルール** | スコア変更後、再計算済み行の **スコア** 列が変わる。 |

### 使い分けの例

- 朝の定点確認 … **概要**
- 「昨日の error でスコア 50 以上」… **イベント** で期間＋重大度＋最小スコア
- 調査結果をチケットに貼る … **運用メモ** ＋ **CSV**

## 7. 画面別操作

### イベント（メインタブ）

画面例: [イベントタブ](../images/events.png)、[ガイド展開](../images/events-event-type-guide-expanded.png)

#### 一覧の更新

- 絞り込み・ページ・表示件数を変えると **自動で再取得** します。
- **設定 → 一般** の **自動更新** がオンで、**イベント** タブを表示している間は、間隔（既定 5 分）ごとにも再取得します（[summary.md](summary.md) §6 と同じ設定）。

#### CSV をダウンロード

- **現在の絞り込み条件**（最小スコア・期間・テキスト 4 種）に一致する行を **すべて** 取得します（ページングは無視）。
- 内部では 200 件ずつ API を繰り返し、最後に **時刻の古い順** に並べ替えて CSV を生成します。
- 列には id、時刻、vCenter 名、種別、重大度、メッセージ、ユーザー名、エンティティ、スコア、タグ、運用メモ などが含まれます。
- 件数が非常に多いと時間がかかります。まず期間や最小スコアで絞ってから実行することを推奨します。

## 8. よくある質問と切り分け

### 一覧が空／件数が想定より少ない

1. **表示期間** が狭すぎないか（または意図せず空＝全期間で他条件だけ効いているか）。
2. **最小スコア** が高すぎないか。
3. テキスト絞り込みの **部分一致** が厳しすぎないか。
4. vCenter **収集** が動いているか（管理者）。

### スコアを変えたのに一覧が古い

- **スコアルール** 保存後は DB 再計算。タブの **自動更新** を待つか、条件を少し変えて再読み込み（[score-rules.md](score-rules.md)）。

### CSV と画面の行数が違う

- 画面は **1 ページ分**（表示件数まで）。CSV は **条件一致の全件** です。

### 運用メモが概要に出ない

- 概要は **再取得時** にサーバーから読みます。保存直後は **イベント** タブで保存済みか確認し、概要タブを開き直すか自動更新を待ちます。

### vCenter ごとに絞りたい

- 現行の **イベント** タブ UI には vCenter プルダウンはありません。種別・メッセージ・CSV 後処理、または **チャット** の対象 vCenter で対応してください。

### アラートが出ているのに一覧に無い

- **評価ウィンドウ** 外・**最小スコア** ・期間フィルタを確認（[alerts.md](alerts.md) §8）。
