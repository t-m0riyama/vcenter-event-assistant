# 概要タブの使い方（利用者向け）

vCenter Event Assistant の **概要** タブは、環境全体を **直近 24 時間** の切り口でざっと把握するためのダッシュボードです。登録 vCenter 数・イベント件数・スコアの高いイベント・多発している種別・負荷の高いホストを、1 画面にまとめて表示します。本書は開発者向けではなく、表示の意味と関連設定を把握したい運用担当者向けに、現行の挙動をまとめたものです。

---

## 1. この文書の対象と読み方

| あなたの役割 | まず読む節 | 目的 |
|-------------|-----------|------|
| **運用（定点確認）** — 毎朝の状況把握 | [§3](#3-上部の統計カード) → [§4](#4-各セクションの見方) | 数字と表の読み方 |
| **運用（深掘り）** — 気になる行を追う | [§4](#4-各セクションの見方) → [イベント](events.md) | 一覧でのフィルタ・運用メモ編集 |
| **設定** — 何が一覧に出るか調整 | [§6](#6-設定一般との関係) | 最小スコア・自動更新 |

用語の区別:

| 用語 | 意味 |
|------|------|
| **直近 24 時間** | サーバーが API を実行した時刻から **遡った 24 時間**（UTC 基準）。暦日（0:00〜24:00）とは限らない。 |
| **要注意（スコア≥40）** | 要注目スコアが **40 以上** のイベント件数（24h 内）。ダイジェスト先頭の同項目と **同じ閾値**。 |
| **要注意イベント（上位）** | スコア順の **最大 10 件** のイベント行（下限スコアは設定で変更可）。 |
| **要注目スコア** | イベントの重要度（0〜100）。**スコア** 列で表示（[score-rules.md](score-rules.md)）。 |

---

## 2. 概要タブの全体像

1. バックエンドが DB から **直近 24h** のイベント・メトリクスサンプルを集計する。
2. **概要** タブを開いているブラウザが `GET /api/dashboard/summary` を呼び、結果を表示する。
3. **設定 → 一般** の **要注意イベントの最小スコア** が、上位一覧に載る行の下限になる。
4. **自動更新** がオンで **概要** タブを表示している間は、一定間隔で再取得する（§6）。
5. 詳細調査は **イベント**・**グラフ**・**チャット** など他タブへ進む。

```
[取り込み済みデータ（DB）]
        ↓
[直近24h で集計] ← 要注意最小スコア（ブラウザ設定）
        ↓
[概要タブに表示]
        ↓
    ┌───┴───┬──────────┬─────────┐
    ↓       ↓          ↓         ↓
[イベント] [グラフ] [チャット] [ダイジェスト]
```

## 3. 上部の統計カード

画面上部の 3 つの数字は、いずれも **直近 24 時間**（§1）の集計です。

| 表示 | 意味 |
|------|------|
| **登録 vCenter** | **設定 → vCenter** に登録されている接続先の件数（有効・無効は問わない）。 |
| **24h イベント** | 直近 24h に取り込まれた **イベント総数**。 |
| **24h 要注意（スコア≥40）** | 直近 24h で要注目スコアが **40 以上** のイベント **件数**（行の一覧ではなくカウント）。 |

「要注意」が増えたときは、下の **要注意イベント（上位）** と **イベント種別（直近24h 件数上位）** をあわせて確認してください。

## 4. 各セクションの見方

各ブロックは **折りたたみ（details）** です。見出し右のプレビュー（例: `3 件`）で件数の目安が分かります。

### 要注意イベント（上位）

| 列 | 内容 |
|----|------|
| **時刻** | イベント発生時刻（**設定 → 一般** のタイムゾーン表示）。 |
| **種別** | `event_type`。 |
| **ガイド** | **表示** でイベント種別ガイド（意味・対処など）を展開。 |
| **スコア** | 要注目スコア。 |
| **メッセージ** | vCenter 由来のメッセージ本文。 |
| **運用メモ** | [イベント](events.md) タブで付けたコメントの **参照のみ**（概要では編集不可）。 |

- 一覧に載るのは **最大 10 件**。スコア降順・同スコアは新しい時刻優先。
- **どのスコア以上を載せるか** は **設定 → 一般** の **要注意イベントの最小スコア**（0〜100、**このブラウザの localStorage**、既定 **1**）。**0** は下限なし（スコア 0 も含む）。
- 種別ガイドで **対処が必要** の行は、背景が強調表示されます（[event-type-guides](../event-type-guides/README.md)）。

### イベント種別（直近24h 件数上位）

| 列 | 内容 |
|----|------|
| **順位** | 件数の多い順（最大 **10 種別**）。 |
| **種別** | `event_type`。 |
| **ガイド** | 種別ごとのガイド（あれば **表示**）。 |
| **件数** | 24h 内の発生回数。 |
| **スコア** | その種別の **最大** 要注目スコア（max notable）。 |

件数は多いがスコアが低い種別（ルーティン操作など）の把握に向きます。

### 高 CPU ホスト / 高メモリ使用率ホスト（直近24h サンプル上位）

2 つのセクションに分かれていますが、列の意味は同じです。

| 列 | 内容 |
|----|------|
| **vCenter** | 接続先の表示名。 |
| **ホスト** | ESXi ホスト名。 |
| **CPU % / メモリ %** | 24h 内サンプルの **ピークに近い上位**（各最大 **10 件**）。 |
| **時刻** | そのサンプルの時刻。 |

メトリクスは `quickStats` 由来のホスト利用率です。**グラフ** タブで時系列を確認し、概要のピークと突き合わせてください。

## 5. 他機能との関係

| 機能 | 概要との違い |
|------|----------------|
| **イベント** | 任意期間・フィルタ・**運用メモの編集**・CSV（[events.md](events.md)）。概要は **24h 固定** の抜粋。 |
| **グラフ** | メトリクス時系列（[graph.md](graph.md)）。概要のホスト表は **24h ピーク上位** のみ。 |
| **ダイジェスト** | **保存済み** の日次/週次/月次レポート。概要は **常に直近24h**（[digests.md](digests.md)）。 |
| **チャット** | 任意期間の対話分析（[chat.md](chat.md)）。 |
| **アラート** | メール通知の履歴。概要は通知を送らない（[alerts.md](alerts.md)）。 |
| **スコアルール** | スコアを変えると、**次回の概要再読み込み後** の数字・一覧に反映。 |

### よくある比較

- **「24h 要注意（スコア≥40）」が 5 件なのに、上位一覧が 10 行ある**  
  上位一覧は **最小スコア 1 以上** なら 40 未満も載り得ます。統計の 40 以上とは切り口が異なります。
- **ダイジェストの要注意件数と概要が一致しない**  
  ダイジェストは **暦日・週・月** の期間。概要は **ローリング 24h** です。

## 6. 設定（一般）との関係

**設定 → 一般** のうち、概要と直接関係する項目です（いずれも **このブラウザの localStorage**）。

| 設定項目 | 概要への影響 |
|----------|----------------|
| **要注意イベントの最小スコア** | **要注意イベント（上位）** に載る行の `notable_score` 下限。変更後は **概要タブを開き直す** か、自動更新を待つと API が新しい値で再取得される。 |
| **タイムゾーン** | 表の **時刻** 列の表示。集計ウィンドウ（24h）自体は UTC ローリングのまま。 |
| **自動更新** / **間隔（分）** | **概要・イベント・グラフ** の各タブを **表示している間**、既定 **5 分** 間隔でデータを再取得（既定はオン、1〜300 分）。別タブにいる間は概要だけのタイマーは動かない。 |
| **チャットの最大保持件数** | 概要には無関係（チャット用）。 |

vCenter の登録・収集の有無は **登録 vCenter** 数と、24h 内にデータが入っているかに表れます。取り込み設定は **設定 → vCenter** とサーバー管理者向けドキュメントを参照してください。

## 7. 画面別操作

### 概要（メインタブ）

画面イメージは [概要タブ](../images/summary.png) を参照してください。

- 初回表示時は「読み込み中…」ののち、統計と各セクションが表示されます。
- エラー時は「概要を読み込めませんでした…」— 画面上部のエラーメッセージと、API・ネットワーク・DB を管理者に確認。
- 各セクションの見出しをクリックして **開閉** できる（既定で「要注意イベント」「高 CPU」「高メモリ」は開いた状態）。
- **運用メモ** を付けたい場合は、該当イベントを [イベント](events.md) タブで検索して編集する。

明示的な「更新」ボタンはありません。**自動更新**（§6）または **タブの切り替え**（他タブから概要に戻ると再読み込み）で最新化されます。

## 8. よくある質問と切り分け

### 数字がずっと 0 のまま

1. **設定 → vCenter** で接続が登録・有効か。
2. 収集ジョブまたは手動取り込みが動いているか（管理者）。
3. 直近 24h に実際にイベントが無いだけの可能性。

### スコアを上げたのに上位一覧が変わらない

- **スコアルール** 変更後、DB の再計算が走る。概要は **次回 API 取得時** に反映。タブを切り替えるか自動更新を待つ（[score-rules.md](score-rules.md)）。

### ガイド列が「—」

- その `event_type` に **イベント種別ガイド** が未登録。 **設定 → イベント種別ガイド** で追加可能。

### 概要とイベント一覧の件数が合わない

- 概要は **24h・上位10件・最小スコア** など **条件が異なる**。**イベント** タブで期間とフィルタを合わせて確認。

### 別の PC では最小スコア設定が違う

- **正常** です。最小スコアと自動更新は **ブラウザごとの localStorage** です。
